01- Fundamentos de git

  • En git un REPOSITORIO es un PROYECTO
  • Cuando aparece una U (UNTRACK) al costado de los archivos del proyecto quiere decir que GIT NO LE ESTA DANDO SEGUIMIENTO

Configuracion Mis datos de usuario

  1. git config --global user.name "Angelo Arevalo"
  2. git config --global user.email "aangelomarcial@gmail.com"
  3. git config --global -e (esto nos mostrara nuestros datos)

Teclas que nos ayudan en Git

  • Q (nos ayda a salir de documnetacion, etc)
  • :wq!Esto nos permite salir cuando estamos en el comando git config --global -e
  • :q!lo mismo de arriba pero aqui solo es para verificar lo que tengamos en la config globañ

Primeros Comandos de Git

  • git --version (El -- significa que viene una palabta completa
    Un - significa que lo que venga despues sera una abreviatura)
  • git help (Nos muestra la ayuda de git)
  • git help commit (Muestra informacion en especifico sobre el commit)

MI PRIMER REPOSITORIO

  1. git init (Inicializa nuestro repositorio local) Nos crea una carpeta .git la cual contiene la referencia a todo lo que vayamos
    haciendo en el repositorio para que git pueda llevar un rastro a todas las modificaciones que vamos haciendo en el repositorio
  2. git status (Nos dara informacion sobre nuestro commit, que es lo que se modifico y que archivos estan listos para su snapshop sobre la rama donde trabaja(main) )
  3. git add (Dara seguimiento a nuestro proyecto)
  4. git add index.html (Le podememos dar seguimiento manualmente al archivo que querramos)
  5. git add . (Toma a todos los archivos del proyecto y les da seguimiento)
  6. git reset .(Aqui va el nombre del archivo) (SACARLO DEL STAGED)Nos permitira remover cualquier archivo que no querramos darle seguimiento(SIEMPRE Y CUANDO YA LE HAYGAMOS DADO SEGUIMIENTO)
  7. git commit -m "Primer commit" (Con esto le damos nombre a nuestro commit)

¿QUE HACE GIT POR NOSOTROS EN ESTOS MOMENTOS

  • git checkout -- . (Reconstruye el proyecto como estaba en el ultimo commit {solo a los archivos que se le esta dando seguimiento }

CAMBIAR NOMBRE DE LA RAMA MASTER A MAIN

  • git branch Nos dice en que rama estamos trabajando
  • git branch -m master main (Esto cambia el nombre de la rama de master a main)

    • ESTABLECER LA CONFIGURACION INICIAL DE QUE AHORA TODAS LAS RAMAS CUANDO VUELVA
    A INICIALIZAR MIS PROYECTOS YA NO SE VA A LLAMAR MASTER SI NO MAIN

  • git config --global init.defaultBranch main

DEMOSTRACION DE LA CREACION PUESTA EN ESCENA Y COMMITS

  • README.md (Es un archivo especial con la extencion md(mackdown) es muy utilizado especialmente al momento de sbir nuestros repositorios a GTIHUB
    las cuales trabajan en base archivos md para colocar informacion visual en ciertas paginas. Y si ponemos en nombre README cuando alguien navegue en su repositorio va a observar todo su contenido del archivo README pero ya formateado de manera html )

    • PARA NO HACER EL git add . , git commit etc,( HAY UN ATAJO PARA HACER COMMIT DIRECTAMENTE LO PODEMOS HACER CON EL SIGUIENTE COMANDO)

  • git commit -am "Readme actualizado" (Este comando solo funciona alos archivos que se le esta dando seguimiento)


  • git log Nos mostrara todos nuestros log que hemos hecho en nuestro repositorio Y ESTARAN ORDENADOS DEL MAS RECIENTE AL MAS ANTIGUO

BONUS- VSCODE ADD Y COMMITS

    Entrar en SOURCE CONTROL

  • Cada vez que se esta modificando en un proyecto aparece en source control (M)podemos realizar por medio del VSCode el git add. y git commit

    • GIT ADD .

  • Al costado del nombre de los archivos aparecen 3 iconos
    1. El primero es para abrir el archivo
    2. El segundo es para descartar cambios que hemos hecho en nuestro archivo
    3. El tercero es como el git add . (SE AGREGA AL staged) con esto ya podremos hacer el commit
    4. Despues de agregarlo al staged nos sale dos iconos (EL DE ABRIR ARCHIVO Y SACARLO DEL STAGED)

    • GIT COMMIT

  • Source control en la parte de arriba hay un rectangulo Se pone el nombre que quieres ponerle al commit y lo guardas con (Ctrl+Enter)

DIFERENTES FORMAS DE AGREGAR ARCHIVOS AL ESCENARIO

  • gid add *.html ( de esta forma podemos agregar todos los archivos HTML al staged)
  • gid add js/*.js ( de esta forma podemos agregar todos los archivos de la carpeta JS al staged)
    • js/ es el nombre de la carpeta
    • *.js es el nombre de los archivos que querramos agregar al staged

    • CUANDO AGREGAMOS UNA CARPETA DENTRO DE NUESTRO PROYECTO Y YA ESTAMOS USANDO GIT DENTRO DE ELLA, GIT NO LO TOMA EN CUENTA
    POR EJEMPLO LA CARPETA ( uplpads AQUI SE VAN A SUBIR TODO LO QUE LOS USUARIOS CARGAN ASU SITIO WEB Y ESTA CARPETA ES MUY IMPORTANTE PORQUE PUEDE QUE EL PAST NO SE CUMPLA Y NO SE PUEDA SUBIR NADA Y EL REPOSITORIO ESTE DAÑADA)PARA SOLUCUINAR ESO SE AGREGA UN ARCHIVO A LA CARPETA CON EL NOMBRE (.gitkeep)

DIFERENTES FORMA DE AÑADIR STAGED

  • gid add css/ (Añade todos los directorios, subdirectorios y archivos que esten dentro de la carpeta CSS)

CREANDO ALIAS PARA NUESTROS COMANDOS

  • git status --short Esto nos mostrara la informacion resumida del git status
  • git config --global alias.s "status --short" Esto creara un alias con el nombre S que remplaza el status --short
  • git config --global alias.lg "log --graph --abbrev-commit --decorate --format=format:'%C(bold blue)%h%C(reset) - %C(bold green) (%ar)%C(reset) %C(white)%s%C(reset) %C(dim white)- %an%C(reset)%C(bold yellow)%d%C(reset)'--all" (Esto es el alias de git log) A git lg
