Git Stash y Git Rebase - Para realizar cambios de emergencia

Introduccion Al Stash

es como una caja o contenedor donde podemos poner todos los cambios temporales para dejarlo en un determinado punto. Se recomienda trabajar solo con un stash y debemos aconstumbrarnos a trabajar con ramas

  • git stash salva todos los directorios aquellos que se le esta dando seguimiento y inclusive a los que no se le esta dando seguimiento,(nos dara un hash para identificarlo)
  • git stash list Podemos ver todos nuestros git stash
  • git stash pop esto va a tomar el ultimo stash y va a colocar nuevamente los cambios que hicimos
  • git stash clear Borra todos los stash
  • git stash drop Elimina el ultimo stash
  • git stash drop stash@{1} De esta forma eliminamos el stash del orden que queramos
  • git stash show stash@{1} Esto nos muestra informacion del stash desde la terminal
  • git stash save "Aqui el nombre o mensaje para tu stash" con esto podemos darle el nombre al stash
  • git stash list --stat Esto nos da un poco mas de info de nuestros stash

INTRODUCCION AL GIT REBASE

REBASE INTERACTIVO

¿Para qué nos serviría eso?

  • Ordenar commits
  • Corregir mensajes de los commits
  • Unir commits
  • Separar commits

REBASE ACTUALIZANDO UNA RAMA (rebase normal)

Si queremos hacer un REBASE tenemos que estar en la rama en la cual quiero que se muevan los cambios ejemplo: seria en la rama 2 y no en main

  • git rebase main tendremos que estar en la rama 2 para poner este comando
  • git merge rama2 con esto unimos los commits de la rama 2 al main(obviamente estando ya en la rama main)

REBASE-SQUASH

  • git rebase -i HEAD~4 el HEAD es el ultimo commit y ~4 indica los ultimos 4 commit Y NOS ABRIRA UNA PG CON DATOS DE LOS COMMITS AGREGADOS Y LUEGO SE TIENE QUE HACER EL SQUASH
  • squash sirve para unir commits
  • Le ponemos solo una (s) al costado del commit solo se pone en un commit cuando quierews unir 2 commit para salir presionamos :qw! y NOS MOSTRARA OTRA PAGINA para salir presionamos :qw!

REBASE - REWORD

CAMBIANDO LOS NOMBRES DEL COMMIT

  1. git rebase -i HEAD~4 elegimos el commit que queremos cambiar el nombre
  2. Despues le ponemos una (r) para cambiarle el nombre del commit

REBASE EDIT

    Podemos hacer un reset y volver hacerlo pero esta ves los haremos con rebase interactivo

  • git rebase -i HEAD~3 en el pick cambien por la letra (e) y salimos :wq! esto nos mandara a un rebase manual sale 2 opciones:
