INTRODUCCION RAMAS, UNIONES Y CONFLICTOS

  • RAMA es una línea de tiempo de commits
  • Merge - uniones
    • Fast-forward se dispara cuando git detecta que no hay ningun cambio en la rama principal y los cambios pueden ser reintegrados de forma trasparente y cada uno de los commits formara parte de la rama principal como si nunca se hubieramos separados (ESTA CARACTERISTICA SE PUEDE DESATIVAR SI NO LA QUEREMOS)
    • Uniones automáticas git detecta que en la rama principal hubo un cambio que la ramass secundarias desconocen, pero cuando intentemos hacer el merge, git dice a ok no se modificaron lineas iguales no hay conflictos lo puede unir de forma automatica y git lo hace sin ningun problema marcando que ambas ramas se unieron
    • Manual Git no puede resolver de forma automatica ejemplo: supongamos que tengamos 2 ramas llamadas 1 y 2 , en la rama 2 hicimos modificaciones que tmb afectan en las mismas lineas en los archivos en la rama que queremos unir los cambios, cuando git intente hacer esto,git entrara en un modo de conflicto en lo cual git nos pedira que nosotros lo resolvamos de forma manual y una vez resuelto el conflicto git creara un commit nuevo que se conoc como (merge commit )

MERGE: FAST-FORWARD

    git branch (Aqui va el nombre que queremos ponerle) de esta forma creamos una rama
  • git checkout (Aqui el nombre de la rama que creamos) Con este comando nos cambiamos de rama

    • CUANDO QUERAMOS UNIR NUESTRA RAMA MASTER Y LA RAMA 1 TENEMOS QUE ESTAR EN LA RAMA QUE QUERAMOS QUE TENGA TODA LA INFO POR EJEMPLO: TENEMOS QU ESTAR EN LA RAMA MASTER

  • git merge (el nombre de la rama que queramos unir (rama1)) De esta forma se unira la rama1 a la rama master

    • CUANDO YA HAYGAMOS UNIDO LA RAMA1 Y NO LA VAMOS A USAR LO CORRECTO SERIA ELIMINARLA

  • git branch -d (el nombre de la rama) De esta forma eliminamos la rama
  • git branch -d (el nombre de la rama) -f Con esto git lo elimara de forma forzada lo recomendable es hacer el de arriba

    • MERGE: UNION AUTOMATICA

    • git checkout -b (nombre de la nueva rama) con este comando creamos una nueva rama y nos mueve a esa rama

    TAGS-ETIQUTAS

    Son una referencia a un commit específico

    Creando etiquetas Tags

    • git tag (el nombre que le vamos a dar al tag) De esta forma creamos tag
    • git tag Nos muestra todo los tags que tengamos
    • git tag -d (Nombre del tag) Con esto eliminamos el tag
    • git tag -a v0.1.0 (aqui el HASH osea el numero del commit) -m "Version Alpha de nestra app"
    • git show (aqui el nombre del tag) Esto nos mostrara informacion del tag
